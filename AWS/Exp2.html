üìù PROCEDURE

  1. Create an S3 Bucket
    • Open the AWS Management Console and navigate to S3.
    • Click Create bucket and enter a unique name (e.g. kisu-first-bucket).
    • Select a region and uncheck Block all public access.
    • Confirm and click Create bucket.
  2. Upload Website Files
    • Open your new S3 bucket.
    • Click Upload and select your index.html and other website files.
    • Click Upload to store them in the bucket.
  3. Enable Static Website Hosting
    • Go to the Properties tab of your bucket.
    • Scroll to Static website hosting ‚Üí Click Edit.
    • Select Enable, and set the index document as index.html.
    • Save changes and note down the Bucket website endpoint URL.
  4. Configure Public Access via Bucket Policy

    Navigate to the Permissions ‚Üí Bucket Policy section and paste the policy below (replace bucket-name appropriately):

    { "Version": "2012-10-17", "Statement": [ { "Sid": "PublicReadGetObject", "Effect": "Allow", "Principal": "*", "Action": "s3:GetObject", "Resource": "arn:aws:s3:::kisu-first-bucket/*" } ] }

    Save the policy and verify that the files are now publicly accessible.

  5. Test Your Static Website
    • Copy the Website Endpoint from your bucket properties.
    • Paste it in your browser (e.g. http://kisu-first-bucket.s3-website.eu-north-1.amazonaws.com)
    • Your website should now be live.
  6. Configure Lifecycle Management
    • Go to your bucket ‚Üí Management tab.
    • Click Create lifecycle rule.
    • Name it (e.g. ArchiveOldFiles).
    • Choose to apply to all objects.
    • Add transitions:
      • Move to Standard-IA after 30 days
      • Move to Glacier after 90 days
      • Delete permanently after 365 days
    • Click Save.
